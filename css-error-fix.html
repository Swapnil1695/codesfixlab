CSS Error: Flexbox Not Centering Vertically

Last updated: November 2023 | Difficulty: Beginner

Error Explanation

Flexbox centering issues are among the most common CSS problems developers face. When using `display: flex` to center elements, you need to apply the correct properties to the parent container. The most common mistake is applying centering properties to the child elements instead of the parent container, or forgetting to set the container height.

For vertical centering to work properly, the parent container must have a defined height. Without a height, the container collapses to the height of its content, making vertical centering appear to not work. This is especially important when trying to center elements in the viewport or within a specific section.

Another common issue is using incorrect property values. For example, using `align-items: center` instead of `justify-content: center` for horizontal centering, or vice versa. Understanding the difference between the main axis and cross axis in flexbox is crucial for proper centering.

Common Causes

  • Applying centering properties to child elements instead of parent
  • Missing container height (collapsed to 0 or auto)
  • Confusing justify-content and align-items properties
  • Using incorrect flex-direction for the desired centering
  • Multiple conflicting CSS rules overriding each other

Step-by-Step Fix

  1. Check the parent container: Ensure the element you want to center is inside a container with `display: flex`.
  2. Set container dimensions: Give the container a defined height (e.g., `height: 100vh` for full viewport).
  3. Apply correct properties: Use `justify-content: center` for main axis (horizontal by default) and `align-items: center` for cross axis (vertical by default).
  4. Check flex-direction: If using `flex-direction: column`, the axes swap - justify-content becomes vertical, align-items becomes horizontal.
  5. Inspect with DevTools: Use browser DevTools to check computed styles and identify conflicting rules.
  6. Test responsiveness: Ensure centering works at different screen sizes.

Code Examples

Error Code (not centering):

CSS - Incorrect 
/* Container has no height */
.container {
    display: flex;
}

/* Centering applied to wrong element */
.child {
    justify-content: center;
    align-items: center;
}

Corrected Code:

CSS - Correct 
/* Container with defined height and centering */
.container {
    display: flex;
    justify-content: center;  /* Horizontal centering */
    align-items: center;      /* Vertical centering */
    height: 100vh;            /* Essential for vertical centering */
}

/* Child doesn't need centering properties */
.child {
    /* No centering properties needed here */
}

FAQ

What's the difference between justify-content and align-items?

justify-content aligns items along the main axis (horizontal by default), while align-items aligns items along the cross axis (vertical by default). If you change flex-direction: column, the axes swap.

Why does my flex container need a height?

Vertical centering requires space to center within. Without a defined height, the container collapses to the height of its content, leaving no extra space to center within. Common solutions: height: 100vh (full viewport), min-height: 100vh, or a fixed pixel height.

Can I center both horizontally and vertically with one property?

Yes! Use place-items: center (shorthand for align-items and justify-items) in CSS Grid, or place-content: center in Flexbox (shorthand for justify-content and align-content).
