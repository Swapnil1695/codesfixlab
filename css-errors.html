CSS Error Fixes

Common CSS layout and styling problems with complete solutions

1. Flexbox Centering Issues

Problem: Items not centering properly

.container {
    display: flex;
}
.center {
    margin: auto;
}

Solution: Use flexbox properties

.container {
    display: flex;
    justify-content: center;  /* horizontal */
    align-items: center;      /* vertical */
    height: 100vh;           /* container height */
}

/* For single item centering */
.single-item {
    margin: auto;
}

Key properties: justify-content, align-items, align-self

2. Responsive Image Problems

Problem: Images breaking layout on mobile

img {
    width: 800px;
    height: 600px;
}

Solution: Responsive image techniques

img {
    max-width: 100%;     /* Never exceed container */
    height: auto;        /* Maintain aspect ratio */
    display: block;      /* Remove extra space below */
}

/* For background images */
.hero {
    background-image: url('image.jpg');
    background-size: cover;
    background-position: center;
    background-repeat: no-repeat;
}

/* Using picture element for art direction */
<picture>
    <source media="(max-width: 600px)" srcset="small.jpg">
    <source media="(min-width: 601px)" srcset="large.jpg">
    <img src="default.jpg" alt="Description">
</picture>

3. Z-Index Not Working

Problem: Elements not stacking correctly

.popup {
    z-index: 9999;
    position: relative;
}

Solution: Understand stacking context

/* z-index only works on positioned elements */
.popup {
    position: fixed;     /* or absolute, relative, sticky */
    z-index: 100;
    top: 0;
    left: 0;
}

/* Parent with lower z-index */
.parent {
    position: relative;
    z-index: 1;         /* Creates stacking context */
}

/* Child cannot appear above parent's stacking context */
.child {
    position: absolute;
    z-index: 9999;      /* Still bound by parent's context */
}

Note: z-index works only with positioned elements (position ≠ static)
